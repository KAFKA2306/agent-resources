# Audit Scope

The audit covers the KAFKA2306 project portfolio at repository level, then performs implementation-level analysis on repositories containing web UI or reusable visual assets.

## Measurements

- project domain and delivery surface
- frontend stack and build system
- color tokens and theme strategy
- typography and density
- spacing, radius, border, and shadow conventions
- navigation and information architecture
- tables, dashboards, cards, forms, and status presentation
- responsive behavior and accessibility signals
- repeated strengths, repeated defects, and outliers

## Evidence rule

A design rule is accepted only when supported by repeated project evidence, an explicit prior preference, or a deliberate corrective decision.
