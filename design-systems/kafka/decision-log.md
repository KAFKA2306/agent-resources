# Decision Log

## D-001 — Do not infer a personal design system from repository names

Repository metadata is sufficient for portfolio classification, but not for visual design inference. The design system must be based on implementation files and rendered interfaces.

## D-002 — Separate inherited style from corrective standards

The final system will distinguish:

1. recurring traits already present across KAFKA2306 projects;
2. explicit aesthetic preferences stated by the owner;
3. corrective rules introduced to remove recurring clutter, weak contrast, excessive spacing, and inconsistent components.
